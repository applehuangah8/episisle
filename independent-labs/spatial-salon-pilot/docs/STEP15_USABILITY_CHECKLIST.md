# Step 1.5 Usability Checklist

Use this list during hands-on validation.

## Core checks

- Can I find project switching without thinking?
- Can I switch World in Project E quickly?
- Can I paste a long messy note and save once?
- Can I see wake-up note immediately after save?
- After refresh, is my latest note still there?

## Timing targets

- Project switch discoverability: <= 10 seconds.
- World switch execution: <= 2 clicks.
- RawPour action completion: <= 8 seconds after paste.
- Re-entry context read: <= 5 seconds.

## Quality guardrails

- No forced formatting before save.
- No data loss when changing project/world.
- No hidden write side-effects in UI components.
- Recovery action available (`Reset pilot data`).
